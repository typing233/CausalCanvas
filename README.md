# CausalCanvas - 交互式知识重构工具

将碎片化文本转化为可视化逻辑网络，并基于连线关系自动生成结构化叙事的交互式知识重构工具。

## 功能特性

- **文本解析**：使用 jieba 中文分词库自动提取文本中的名词、人名、事件
- **标签分类**：将提取的实体按人物、事件、概念分类展示
- **拖拽交互**：支持将标签拖拽到画布成为节点
- **节点连接**：可视化的贝塞尔曲线连线，支持 10 种预设关系类型
- **报告生成**：基于有向图数据自动生成连贯的纯文本总结
- **一键复制**：方便地将生成的报告复制到剪贴板

## 快速开始

### 1. 安装依赖

```bash
cd backend
pip install -r requirements.txt
```

### 2. 启动服务

```bash
cd backend
python main.py
```

或者使用 uvicorn 直接运行：

```bash
cd backend
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

### 3. 访问应用

打开浏览器访问：`http://localhost:8000`

## 使用说明

### 步骤 1: 输入文本

在左侧文本框中粘贴或输入您想要分析的文本内容。

### 步骤 2: 解析文本

点击 **"解析文本"** 按钮，系统将自动：
- 使用 jieba 分词提取关键词
- 识别并分类人物、事件和概念
- 在左侧下方显示提取的标签

### 步骤 3: 筛选标签

使用标签上方的筛选按钮：
- **全部** - 显示所有类型的标签
- **人物** - 只显示人物类标签（蓝色）
- **事件** - 只显示事件类标签（绿色）
- **概念** - 只显示概念类标签（黄色）

### 步骤 4: 创建节点

将左侧的标签 **拖拽** 到中央画布上，松开鼠标即可创建节点。

### 步骤 5: 移动节点

点击并拖动节点可以在画布上自由移动位置。

### 步骤 6: 连接节点

1. 将鼠标悬停在节点上，会显示四个方向的连接点（蓝色圆点）
2. 从一个节点的连接点 **按下并拖动**
3. 松开鼠标到另一个节点的连接点上
4. 在弹出的关系选择器中选择合适的关系类型

### 步骤 7: 编辑关系

点击连线上的关系标签，可以重新选择关系类型。

### 步骤 8: 删除节点/连线

- 悬停在节点上，点击右上角的 **×** 按钮删除节点
- 选中节点后按 **Delete** 或 **Backspace** 键删除
- 删除节点会同时删除相关联的连线

### 步骤 9: 生成报告

点击画布工具栏上的 **"生成报告"** 按钮：
- 系统将分析画布上的所有节点和连线
- 使用预设的模板生成连贯的文本总结
- 报告将显示在右侧面板

### 步骤 10: 复制报告

点击 **"一键复制"** 按钮，报告内容将复制到剪贴板。

## API 接口

### POST /api/analyze

分析文本并提取关键词。

**请求体:**
```json
{
    "text": "要分析的文本内容..."
}
```

**响应:**
```json
{
    "entities": [
        {"text": "张三", "type": "person", "count": 3},
        {"text": "革命", "type": "event", "count": 2},
        {"text": "国家", "type": "noun", "count": 5}
    ],
    "keywords": ["张三", "革命", "国家", "..."]
}
```

### POST /api/generate-report

基于图数据生成报告。

**请求体:**
```json
{
    "nodes": [
        {"id": "node-1", "label": "事件A", "type": "event"},
        {"id": "node-2", "label": "事件B", "type": "event"}
    ],
    "edges": [
        {"source": "node-1", "target": "node-2", "relation": "导致"}
    ]
}
```

**响应:**
```json
{
    "report": "本报告总结了以下关键要素：\n\n关键事件：事件A, 事件B。\n\n---\n\n关系分析：\n\n- 事件A导致了事件B的发生。"
}
```

## 预设关系类型

- **导致** - 直接因果关系
- **造成** - 造成某种结果
- **导火索** - 触发事件
- **影响** - 产生影响
- **促使** - 促使发展
- **引发** - 引发事件
- **推动** - 推动进程
- **阻碍** - 阻碍发展
- **起源于** - 来源关系
- **结果是** - 结果关系

## 快捷键

- `Delete` / `Backspace` - 删除选中的节点
- `Escape` - 取消连接操作 / 取消选中

## 开发说明

### 修改前端

前端文件位于 `frontend/` 目录下：
- `index.html` - 页面结构
- `style.css` - 样式定义
- `app.js` - 应用逻辑

### 修改后端

后端文件位于 `backend/` 目录下：
- `main.py` - FastAPI 应用和 API 路由

### 添加新的关系类型

1. 在 `backend/main.py` 的 `relation_templates` 字典中添加新模板
2. 在 `frontend/index.html` 的关系选择器中添加对应按钮
3. 在 `frontend/app.js` 中确保处理逻辑兼容
